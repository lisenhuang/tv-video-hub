Step 1 — Connect a database

Only the database connection is stored on this server. Your admin account, storage config and release key live in the database. Choose a provider, then Test & continue.

How to get these from Cloudflare D1
  1. Create the database. Cloudflare dashboard → Storage & Databases → D1 SQL Database → Create. Open it and copy its Database ID into the D1 Database ID field below.
  2. Account ID. Same dashboard — it's in the right-hand sidebar, or the hex in the URL dash.cloudflare.com/<account-id>. Paste it into Account ID.
  3. API Token. Top-right profile menu → My Profile → API Tokens → Create Token → Create Custom Token. Add the permission Account › D1 › Edit, save, and copy the token into D1 API Token.
  4. Click Test & continue — the backend creates its tables automatically on first connect.

Object storage
How to get these from Cloudflare R2
  1. Create a bucket. Cloudflare dashboard → R2 Object Storage → Create bucket. Copy the bucket name into Video bucket below (e.g. tv-videos).
  2. S3 endpoint & region. R2 → Overview (or a bucket → Settings) shows the S3 API URL https://<account-id>.r2.cloudflarestorage.com. Paste it into Service URL and set Region to auto.
  3. API token = access keys. R2 → Manage R2 API Tokens → Create API Token. Permission Object Read & Write (scope it to your bucket[s]). On creation Cloudflare shows an Access Key ID and Secret Access Key once — copy them into the matching fields here.
  4. R2 needs these toggles on: Force path-style addressing, Disable upload payload signing, and Checksums only when required. Use the button below to set the region + all three at once.
  5. Click Save settings, then Test connection — it writes a tiny probe object to confirm the bucket is reachable. The app's playback URLs are short-lived presigned R2 URLs, so the bucket can stay private.
